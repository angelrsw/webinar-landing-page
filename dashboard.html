Zoom

Not connected

Create a Zoom Server-to-Server OAuth app in your Zoom Marketplace with scopes webinar:read:admin and webinar:write:admin. Paste the credentials into the Worker's environment variables — do not put them in this browser.

GitHub (landing page repo)

Not connected

The Worker will commit an updated webinars.json to angelrsw/webinar-landing-page using a fine-scoped Personal Access Token. PAT is stored as a Worker secret, never in the browser.

How this works

Short walkthrough of the flow so you know what's ready today vs. what's coming in Phase 2.

Phase 1 · today

Live
  1. Log attendee requests after each webinar.
  2. See which topics are most-requested on the Overview.
  3. Create a webinar in Zoom, paste the registration URL here, and publish it to the landing page (locally for now).
  4. Edit or remove webinars anytime.
  5. Export a JSON backup whenever you want.

Phase 2 · to enable automation

Planned
  1. Cloudflare Worker + D1 database deployed.
  2. Zoom Server-to-Server OAuth app connected — enables "create in Zoom" in one click.
  3. GitHub PAT stored as a Worker secret — the Worker commits webinars.json to the repo for you.
  4. Landing page updated to read webinar blocks from webinars.json so it refreshes automatically.
  5. Optional: simple password login for the dashboard.
